Overview

This assignment helps you solidify your understanding of modern HTML5 structure while applying foundational concepts of web accessibility and SEO. You will create a semantically correct web page that prioritizes both human and machine readability.

Objective

Build a basic web page using HTML5 semantic tags, applying accessibility best practices and beginner-friendly SEO principles. The final output should demonstrate a well-structured layout that supports screen readers and is optimized for discoverability.

Guidelines

  • Use only HTML5. No CSS or JavaScript is required.
  • Focus on meaningful semantic elements; avoid <div> or <span> unless necessary.
  • Include clearly defined sections: header, navigation, main content, and footer.
  • Enhance accessibility with proper landmarks, ARIA attributes, and logical navigation.
  • Use heading tags in the correct hierarchy for SEO and readability.

Tips

  • Use semantic HTML5 elements like <header>, <nav>, <main>, <section>, and <footer> appropriately.
  • Organize content for accessibility, including screen reader navigation and descriptive links.
  • Apply basic on-page SEO: meaningful headings, metadata, descriptive text.
  • Maintain clean, readable HTML structure for both humans and search engines.

Deliverables

  • A single HTML file named index.html.
  • Semantic structure using appropriate HTML5 elements.
  • Clear headings in logical order.
  • Accessibility enhancements with proper tags and attributes.
  • SEO-friendly metadata and content.
